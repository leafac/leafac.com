Pre-Requisites

First-class continuations & call/cc. This article is only about the type of call/cc, not about what it is or how it works.

Reading code written in Racket. Racket is used only as the language of discourse; the results of the article are about type theory and apply to other languages.

Note

This type is interesting, particularly in its connection to logic, but like most ideas related to first-class continuations, it’s unintuitive. In this article we derive the type of call/cc step by step using examples written in Racket. Racket is dynamically typed, but we may use it to reason about static types as well.

Note

There’s a version of Racket that is statically typed, Typed Racket, but the type system in Typed Racket is more sophisticated than a classical Hindley–Milner type system, so the type of call/cc in Typed Racket is more sophisticated as well (see appendix).

Then we consider the following expression:

> (zero? 0)
 
#t

We may surround the 0 in this expression with an use of call/cc while preserving its output:

> (zero? (call/cc (λ (k) 0)))
 
#t

The output is preserved because call/cc calls (λ (k) 0) with the current continuation, but (λ (k) 0) ignores its parameter and returns 0.

Note

In general, for any expression e, the result of computing e is the same as the result of computing (call/cc (λ (k) e)) as long as e doesn’t refer to k.

Note

The type of k is , so k has the type of a function that receives an argument of any type and produces a result of the type you need. If the call to k appears as argument to string-length, for example (string-length (k 29)), then k produces a String; if the call to k appears as argument to zero?, for example (zero? (k "Leandro")), then k produces a Number; and so forth. It makes sense for a type system to behave like this because execution skips over the string-length, zero?, and so forth, so no runtime type error could occur.

This shows that continuations like k don’t behave like regular functions.

Appendix: The Type of call/cc in Typed Racket

This more general form is useful to type programs like the following:

> (write (call/cc (λ (k) (k "Leandro") 0)))

Note

This kind of limitation in classical Hindley–Milner type systems occurs in many situations beyond call/cc. For example, in classical Hindley–Milner type systems the types of the then and else branches of a conditional must agree:

(if <condition> 0 29)

But in Typed Racket their types may be different, for example:

(if <condition> "Leandro" 29)
